# Workflow Overview

BW Atlas follows an operational workflow suitable for inspection-led field projects:

Manager -> Inspection -> Task -> Worker -> Approval -> Report

## 1. Manager
Manager creates or owns project context, assigns teams, configures workspace scope, and controls task visibility.

## 2. Inspection
Inspection data is captured per module (Fire Door, Fire Stopping, Snagging) with field evidence, statuses, and defect tracking.

## 3. Task
Findings are transformed into actionable tasks (for example remedials, installation, verification), each with clear workflow state.

## 4. Worker
Workers execute assigned tasks, add evidence/progress updates, and submit outcomes back into the controlled workflow.

## 5. Approval
Manager/reviewer validates submitted work, records decisions, and enforces sign-off criteria according to role permissions.

## 6. Report
System generates structured PDF reports and summary outputs for operational handover, audit history, and portfolio evidence.

## Notes for Academic Assessment
The workflow demonstrates:
- role-based access design
- modular domain modeling
- practical end-to-end state transitions
- report generation and traceability
